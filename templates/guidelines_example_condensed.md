## Usage Notes for LLMs

When generating new content:

1. **Start with WHY**: Understand the purpose before writing
2. **Reference examples**: Use the transformations above as models
3. **Apply systematically**: Go through checklist for each piece
4. **Prioritize clarity**: When in doubt, be clearer rather than fancier
5. **Check client_rules.js**: Validate against forbidden words, terminology, patterns
6. **Read aloud**: If it sounds unnatural to a native speaker, revise

**Remember**: These guidelines capture patterns observed in [CLIENT_NAME]'s existing content. When making trade-offs, favor consistency with these established patterns over general best practices.
